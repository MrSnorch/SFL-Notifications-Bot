# 🌻 SFL Farm Notifier

Телеграм-бот для мониторинга ферм [Sunflower Land](https://sunflower-land.com).  
Каждый пользователь настраивает свою ферму прямо в Telegram — уведомления приходят в личку.

**Полностью бесплатно:** публичный GitHub репозиторий = безлимитные минуты Actions.

---

## Архитектура

```
Telegram Bot (команды пользователей)
        ↕
   Supabase (пользователи, настройки, состояние)
        ↕
GitHub Actions (сканер + бот работают 24/7)
        ↕
Telegram DM (личные сообщения каждому пользователю)
```

Два независимых процесса:

- **Bot** — отвечает на команды пользователей, меняет настройки
- **Scanner** — опрашивает API Sunflower Land, шлёт уведомления о готовности

---

## Режимы сканера

### Обычный режим (`scanner.yml`)
Один job обходит всех пользователей по очереди с интервалом 30 секунд между запросами.  
Подходит для небольшого числа пользователей.

### Matrix-режим (`scanner_matrix.yml`)
Каждый пользователь получает **отдельный runner** — свой IP, свой цикл ~5ч 45м.  
Параллельная работа, нет очереди, нет деления полосы между пользователями.

**Дополнительно в matrix-режиме — «Ранний подъём»:**  
После каждого скана бот знает, через сколько секунд созреёт следующий ресурс.  
Если это меньше 30 секунд (интервала между сканами) — runner засыпает ровно до этого момента и шлёт уведомление **без запроса к API**, точно в нужную секунду.  
При 429 (rate limit) — job автоматически перезапускается на новом runner с чистым IP.

## Что получают пользователи

### Закреплённое статус-сообщение
Обновляется молча (без уведомления) при каждом скане:

```
🌻 Ферма 12345

🌾 Sunflower [3] — через 00:45:23 — 14:30
🪵 Trees [5] — через 01:12:00 — 15:00
🍎 Apple [2] — через 3д 12:00:00 — послезавтра

✅ Готово к сбору:
  🌾 Carrot [2/4]

Обновлено: 05.05 13:44
```

### Алерт о готовности
Приходит с уведомлением когда ресурс созрел:

```
🌾 Carrot [4/4] — готово к сбору ✅
```

Счётчик `[готово/всего]` учитывает ресурсы, которые созреют в течение следующих 30 секунд — чтобы не приходило `[1/22]` когда все деревья уже фактически готовы.

Алерт автоматически удаляется когда пользователь собирает урожай.  
Повторные напоминания настраиваются через `/settings` (до 5 раз с интервалом от 5 до 60 минут).

## Установка

### 1. Создать бота в Telegram

1. Написать [@BotFather](https://t.me/BotFather)
2. `/newbot` → дать имя → получить токен вида `7123456789:AAF...`
3. `/setcommands` → вставить список команд:

```
start - Начать / перерегистрироваться
setfarm - Установить ID фермы
setkey - Установить API ключ
settings - Настройки отслеживания
status - Статус фермы прямо сейчас
stop - Приостановить уведомления
resume - Возобновить уведомления
lang - Сменить язык
help - Помощь
```

### 2. Создать базу данных в Supabase

1. Зайти на [supabase.com](https://supabase.com) → New project
2. В **SQL Editor** выполнить содержимое файла `supabase_schema.sql`
3. Скопировать из **Settings → API**:
   - **Project URL** → `SUPABASE_URL`
   - **anon public** key → `SUPABASE_KEY`

### 3. Залить на GitHub

```bash
git init
git add .
git commit -m "SFL Notifier"
git remote add origin https://github.com/ВАШ_АККАУНТ/sfl-notifier
git push -u origin main
```

⚠️ Репозиторий должен быть **публичным** — только тогда GitHub Actions бесплатны.

### 4. Добавить секреты

`Settings → Secrets and variables → Actions → New repository secret`:

| Имя | Значение |
|-----|---------|
| `TELEGRAM_BOT_TOKEN` | Токен от BotFather |
| `SUPABASE_URL` | URL проекта из Supabase |
| `SUPABASE_KEY` | anon key из Supabase |

### 5. Запустить

В **GitHub → Actions** запустить вручную (кнопка «Run workflow»):

- **SFL Bot** — бот для приёма команд
- **SFL Scanner** или **SFL Scanner Matrix** — сканер ферм

После первого ручного запуска оба workflow перезапускаются автоматически через self-restart. Страховочный cron срабатывает раз в 6 часов если self-restart по какой-то причине не сработал.

## Команды пользователя

| Команда | Описание |
|---------|---------|
| `/start` | Регистрация или перерегистрация |
| `/setfarm 12345` | Установить ID фермы (число из URL на сайте) |
| `/setkey abc123...` | Установить API ключ (бот сразу удалит сообщение) |
| `/settings` | Настройки: что отслеживать, повторы, часовой пояс |
| `/status` | Показать текущий статус фермы прямо сейчас |
| `/stop` | Приостановить уведомления |
| `/resume` | Возобновить уведомления |
| `/lang` | Сменить язык (🇷🇺 ru / 🇬🇧 en / 🇺🇦 uk) |
| `/help` | Справка |

### Настройки (`/settings`)

Через inline-кнопки можно управлять:

- **Что отслеживать:** урожай, деревья, камни, железо, золото, кримстоуны, нефть, соль, санстоуны, фрукты, цветы, мёд, животные
- **Повторные напоминания:** количество (0–5) и интервал (5, 10, 15, 30, 60 минут)
- **Часовой пояс:** выбор из списка или ввод вручную (`Europe/Kyiv`, `UTC+3` и т.п.)

## FAQ

**Какой сканер лучше выбрать?**  
Matrix — если больше 2–3 пользователей или нужна точность уведомлений до секунды. Обычный — проще для одного пользователя.

**Есть ли задержка при рестарте?**  
До 10 минут каждые ~6 часов — GitHub Actions запускается не мгновенно. Self-restart минимизирует паузу, cron подстрахует если что-то пошло не так.

**Что если API Sunflower Land недоступен (503)?**  
Сканер логирует ошибку и пропускает итерацию — уведомления не теряются, следующий скан сделает попытку снова.

**Что если Telegram API завис (таймаут)?**  
Закреплённое сообщение остаётся нетронутым — бот не будет слать дубли и не будет заново его перезакреплять. Следующий скан отредактирует его как обычно.

**Пользователи видят данные друг друга?**  
Нет. Каждый API-ключ и состояние привязаны к `telegram_id`. RLS в Supabase защищает строки на уровне базы данных.

**Максимум пользователей?**  
Supabase free tier: 500 МБ — хватит на тысячи пользователей. GitHub Actions для публичных репо: параллельные jobs без лимита.
